# FreeSQL Vector Demo: Car Parts Search

This getting-started lab builds a small semantic search application for automotive parts inventory.

The Python app:

- creates an Oracle table with a `VECTOR(384, int8)` embedding column
- generates 1,000 realistic car-part inventory rows
- creates embeddings with `all-MiniLM-L6-v2`
- stores the vectors in Oracle Database
- exposes a REST API for vector search
- serves a lightweight React UI from the same Python process

## Prerequisites

- Python 3.11 or newer
- An Oracle Database account that supports the `VECTOR` data type
- A FreeSQL or Oracle Database connection string, username, and password

## Setup

Create a virtual environment and install dependencies:

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
```

Create your local environment file:

```bash
cp .env.example .env
```

Edit `.env` with your Oracle connection values:

```bash
ORACLE_USER=your_freesql_username
ORACLE_PASSWORD=your_freesql_password
ORACLE_DSN=(description=(address=(protocol=tcps)(port=2484)(host=your-host.example.com))(connect_data=(service_name=your_service_name)))
```

## Initialize The Lab Table

Create and seed the `AUTOMOTIVE_PARTS` table:

```bash
python app.py --reset --init-only
```

The first run downloads the `all-MiniLM-L6-v2` model and generates embeddings for 1,000 parts. Later runs skip reseeding when the table already has the target inventory size.

## Run The App

```bash
python app.py --host 127.0.0.1 --port 8088
```

The web app can search existing data, install the demo table when it is missing, or reinstall the demo table from the browser. If you want the app to initialize data before serving, run:

```bash
python app.py --auto-init --host 127.0.0.1 --port 8088
```

Open:

```text
http://127.0.0.1:8088
```

Try searches like:

- `Toyota Camry front brake pads quiet`
- `2019 F-150 misfire rough idle coil`
- `overheating coolant leak radiator`
- `weak cabin airflow blower motor`
- `wheel growl ABS sensor hub`

## REST API

Health check:

```bash
curl http://127.0.0.1:8088/api/health
```

Search with query parameters:

```bash
curl "http://127.0.0.1:8088/api/search?q=Toyota%20Camry%20front%20brake%20pads&limit=3"
```

Search with JSON:

```bash
curl -X POST http://127.0.0.1:8088/api/search \
  -H "Content-Type: application/json" \
  -d '{"query":"2019 F-150 misfire rough idle coil","limit":5}'
```

Start a browser-driven install or reinstall job:

```bash
curl -X POST http://127.0.0.1:8088/api/admin/install \
  -H "Content-Type: application/json" \
  -d '{"reset":false}'
```

Check install progress:

```bash
curl http://127.0.0.1:8088/api/admin/install-status
```

## What To Notice

The search uses semantic similarity, not keyword matching. A query like `rough idle weak spark` can find ignition coils even if the exact query text is not stored as a keyword field.

Embeddings are generated by `all-MiniLM-L6-v2` and quantized to `int8` before storage. This keeps the lab small enough for constrained FreeSQL-style environments while still demonstrating Oracle vector search.

The core SQL search pattern is:

```sql
select *
from (
    select
        id,
        part_number,
        name,
        category,
        description,
        unit_price,
        vector_distance(embedding, :query_vector, cosine) as distance
    from automotive_parts
    order by distance
)
where rownum <= :result_limit
```
